## Image Management Strategy

### Git LFS (Large File Storage) for High-Resolution Images

**Strategy:** Dual-image approach for optimal performance and media downloads

**How It Works:**
1. Client uploads ONE high-resolution image via CMS
2. System automatically creates TWO versions:
   - **Optimized web version** â†’ `/public/images/web/` (regular Git)
   - **High-res original** â†’ `/public/images/downloads/` (Git LFS tracked)
3. Website displays optimized images (fast loading)
4. Download buttons provide high-res originals (for media use, press kits)

**Directory Structure:**
```

â”‚   â”œâ”€â”€ headshot-1.jpg           # 800-1200px width, ~200-400KB
â”‚   â”œâ”€â”€ headshot-2.jpg
â”‚   â””â”€â”€ production-1.jpg
â””â”€â”€ downloads/                    # High-res originals (Git LFS)
    â”œâ”€â”€ headshot-1.jpg           # Original size, 2-8MB
    â”œâ”€â”€ headshot-2.jpg
    â””â”€â”€ production-1.jpg
```

**Git LFS Configuration:**
```bash
# Initialize Git LFS
git lfs install

# Track high-res images folder
git lfs track "public/images/downloads/*.jpg"
git lfs track "public/images/downloads/*.png"
git lfs track "public/images/downloads/*.webp"

# Commit .gitattributes
git add .gitattributes
git commit -m "Configure Git LFS for high-res images"
```

**.gitattributes file:**
```

public/images/downloads/*.png filter=lfs diff=lfs merge=lfs -text
public/images/downloads/*.webp filter=lfs diff=lfs merge=lfs -text
```
